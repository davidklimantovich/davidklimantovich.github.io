Module 1: AP CS: Programming

This is perhaps my favorite module of them all.
It is an all-purpose introductory module to JavsScript that teaches the user the fundamentals of the language.

It limits opportunities for hands-on learning through projects and challenges but allows the student to firstly get a good knowledge of concepts so that they are at least able to read and understand code. Having done this module first and continuing on to Intro To JS, it was clear to me that doing this module significantly increased my knowledge and capabilities so that I was able to properly use them in the next module.

Module 2: Intro to JS: Drawing and Animation

As stated in the previous paragraph, this module also let me grasp key concepts for JavaScript. This time, however, I was able to actually incorporate creativity in coding by having projects as benchmarks for my understanding.

It was easily the most enjoyable of all modules, but I still rank it as the 2nd in the race for the best module because although having a module be an enjoyable learning experience is a contributing factor of ranking, it is also important to include modules that have had a significant impact on my understanding of code. That is why I chose to put it at number 2.

Here is an embedded project that was done in this unit: "Make it rain".

Module 3: Advanced JS: Games and Visualization

In the third place, I chose the Advanced JS unit, Games and Visualization. An instant factor that was used in choosing this module was my personal appreciation of video games. I personally want to become a game developer as a career and this module allowed me to be introduced to the world of game design.

It was an incredibly enjoyable experience, and the only reason why I don’t rank it as number two on the list is that I believe that Intro To JS was more valuable to me in terms of learning. And while this module did teach me a lot that I wanted to know, it was Intro To JS that allowed me to get a learning grip about the world of coding. It was the first real introduction I had to coding way back in grade 9 and I owe it to that module to rank it above this.

Here is an embedded project that was done in this unit: "Memory++".

Module 4: Intro To HTML and CSS

The module that I chose for 4th place is nonother than Intro to HTML. This module was the bedrock for my understanding of web development which is another aspect that I am interested in. It had the same learning concepts as Intro to JS; a quick, simple easy introduction to a coding language and aspect that allowed me to cement my understanding and really captivated my attention.

The only reason as to why I am placing it below all the other modules is because, for me, they were much more enjoyable and in my area of interest. However, I will give due credit for this module as having been one of the most informative pieces of learning in this class.

Here is an embedded project that was done in this unit: "Blog".

Module 5: FCC: Basic HTML and HTML5

This module really essentially ties in with the previous one. It is more of an extra learning opportunity than a new module or a continuation of the previous one. It has the same key concepts as the KA version, only this time it is a different format for learning.

The only reason it is not on the same level as Intro to HTML is a personal preference. Other than that, there is not much to say about this module. It is simple, easy to understand, and enjoyable. If you would like to see the project I have done for yourself, heres the link. Here is a image proof of my completion of all the lessons in the module.

Module 6: Live coding refactoring

This module is quite a unique one in that it is taught by a student. I had to refactor a project that was chosen by the presenters and who taught the entire class what to do. This is quite different from what I am used to when learning certain concepts and it was overall a fairly enjoyable task.

The main difference from what it is usually like is that this time, we didn’t necessarily learn any new concepts but more how to apply them. Refactoring a pre-existing project deals more with creativity than functionality and theory. In this sense, I had learned about how to restructure code to fit my personal creative ideas.

Module 7: Advanced JS: Natural Simulations

I have ranked the final module that I have done that has coding aspects in as number 7. This has been the least enjoyable module that had to do with coding. And although it is at number 7, it still beats the three other modules that had nothing to do with coding. The reason I placed it at last place is that this module deals a lot with physics and really complex mathematical concepts.

And while I am prepared to put in the work to learn these complex ideas, it was not as enjoyable or straight forward as the previous ones. And while this could be interpreted as negative, I think it could be positive. Having these new challenges be presented to you could be a new learning experience and can prepare you for the challenges that you can face in the future.

It can also be reassuring, knowing that if you are prepared to put that much effort into a module on Khan Academy will indicate that you won’t be afraid to back down from challenges that you will undoubtedly face in your career as a programmer.

Here is an embedded project that was done in this unit: "Paint splatter".

Module 8: Digital information and Algorithms

For the first non-coding based module, I decided to put the combination of Digital Information and Algorithms. I chose this duo as the best out of the three non-coding based modules because although it isn’t exactly centered around coding, this combination is critical in the world of coding. Understanding algorithmic principles and how information is handled digitally is essentially 50% of what coding is.

This module is unique because it does not focus on a specific aspect of a language but rather a general area that is extremely useful when coding. However, on account of it not being extremely intertwined with coding as the previous modules have been, I have decided to place it in 8th place. This is because my personal criteria on how to rank these modules stem from the fact that I believe that coding oriented modules are much more important to learn in this course than none. I assume that is why this module is not mandatory and thus I personally chose to do it.

Module 9: Simulations, Data analysis and online data security

The second noncoding based module, this combination is focused on data and security. This was a very enjoyable module with a lot of interesting information. Unfortunately, it doesn’t tie in with coding as the previous modules so I had to place it at number 9. However, I could see how others could place it higher.

Data analysis in particular is something I believe is incredibly interesting, however, I do not want to pursue it in the future. Even so, it still is important when creating certain programs. Simulations and online data security are important but do not captivate me as much.

Module 10: Computing innovations and The Internet

In the final place, I chose to put computing innovations and the internet. This module was quite unique as it explores topics typically not associated with computer science. This made it a new experience and certainly a memorable one. It was unlike any of the others, as it something I did know was this complex, considering I use the internet all the time.

However, I can say for certain that this module might have been the most useful for me to apply in the real world, as I learned some information that I can use on a day to day basis considering how integrated the internet is with our lives especially during the COVID-19 pandemic. It was an informative and fairly easy to learn module that had a significant impact on my understanding of the web and the other aspects of life that I use so very often but never really pay attention to.
